# Snowflake Agentic Platform

> From use case description + raw data → deployed AI application

A self-assembling agent system that takes a natural language use case description and raw data assets, then autonomously builds and deploys a complete AI-powered application on Snowflake.

## What It Does

```
INPUT:  "I have sensor data and daily reports. Need to predict equipment 
         failures and search historical incidents."
      + [sensor_readings.parquet, reports/*.pdf]

OUTPUT: - Deployed React + FastAPI app on SPCS
        - Cortex Search service for document retrieval
        - Cortex Analyst semantic model for data queries
        - Registered ML models for predictions
        - Cortex Agent orchestrating all tools
```

## Core Principles

| Principle | Description |
|-----------|-------------|
| **Domain Agnostic** | No pre-built templates. Agents discover and adapt to any domain. |
| **Generated, Not Templated** | App code is generated by LLMs, not filled into templates. |
| **Granular Sub-Agents** | Deep hierarchies with specialized sub-agents for each task. |
| **Self-Healing** | Sophisticated validation with retry loops and error recovery. |
| **Snowflake Native** | All state, compute, and AI services run on Snowflake. |
| **Cortex Maximalist** | Leverage every Cortex capability: LLM, Search, Analyst, Agents. |

## Architecture

```
┌─────────────────────────────────────────────────────────────────────┐
│                         META-AGENT (Planner)                        │
│  Parses use case → Queries Agent Registry → Generates Execution Plan│
└─────────────────────────────────────────────────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────────────┐
│                      EXECUTION PHASES (LangGraph)                   │
│                                                                     │
│  Discovery → Preprocessing → Validation → Feature Store →          │
│  ML Training → Search Index → Semantic Model → App Gen → Deploy    │
│                                                                     │
│  Each phase: Multiple sub-agents, checkpointed, retriable           │
└─────────────────────────────────────────────────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────────────┐
│                         DEPLOYED APPLICATION                        │
│  React + FastAPI on SPCS, powered by Cortex Agent/Search/Analyst   │
└─────────────────────────────────────────────────────────────────────┘
```

See [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) for full architecture details.

## Project Structure

```
snowflake-agentic-platform/
├── src/
│   ├── meta_agent/          # The Planner (LangGraph)
│   ├── agents/              # Sub-agent implementations
│   │   ├── registry/        # Agent capability catalog
│   │   ├── discovery/       # File scanner, schema profiler
│   │   ├── preprocessing/   # Parquet processor, document chunker
│   │   ├── validation/      # Data & semantic validators
│   │   ├── ml/              # ML model builder, feature store
│   │   ├── search/          # Cortex Search builder
│   │   ├── semantic/        # Semantic model generator
│   │   ├── app_generation/  # Code generator, test agent
│   │   └── deployment/      # SPCS deployer
│   ├── state/               # LangGraph state & Snowflake checkpointing
│   ├── observability/       # LangSmith tracing
│   └── app/                 # Orchestrator UI
├── tests/                   # Unit, integration, LangSmith evals
├── deployment/              # SPCS Dockerfile & configs
└── docs/                    # Architecture & guides
```
